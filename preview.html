This page only shows the 7-step information architecture of the quotation builder. Use it for review / presentation. No data entry here.

Step 1 · Project header

Client / project / commercial context

  • • Client name / site / contact
  • • Project title, internal ref
  • • Currency, tax profile, locale
  • • Auto: quote no. & date

Word: <BM:ClientName>, <BM:ProjectTitle>

Step 2 · Site & process

Layout / utilities / constraints

  • • Indoor / outdoor / clearance
  • • Capture points, duct length
  • • Power / noise / zone (ATEX/NFPA)
  • • Auto: base CFM / SP hint

Word: <BM:SiteSummary>, <BM:Constraints>

Step 3 · Performance & compliance

Design targets & safety envelopes

  • • Target CFM / max SP
  • • ATC (air-to-cloth)
  • • Standards (NFPA / CSA / ASHRAE)
  • • Dust / explosion class

Word: <BM:PerfTargets>, <BM:ComplianceFlags>

Step 4 · Options (A/B/C)

Engineering + BOM per option

  • • New / retrofit / modular
  • • Filter / fan / hopper / duct
  • • Validation: CFM ≥ target, filter area OK
  • • Lead-time summary

Word: <BM:OptA_Title>, <TB:OptA_ItemTable>

Step 5 · Costing & pricing

MAT + LAB + logistics + margin

  • • Materials, labour, travel
  • • Discount / contingency
  • • Margin / tax profile
  • • Payment schedule (30/20/30/20)

Word: <BM:OptA_TotalBeforeTax>, <TB:OptA_PaymentSchedule>

Step 6 · Contract / delivery / warranty / scope

Client/company responsibilities

  • • Engineering / fabrication / install weeks
  • • Warranty (1y/2y)
  • • Scope split (company vs client)
  • • Legal clauses

Word: <TB:ScopeCompany>, <TB:ScopeClient>

Step 7 · Preview & export

EN/FR, customer/internal views

  • • EN / FR templating
  • • Hide margin for customer
  • • Word / PDF export
  • • File naming rules

Output: merged document

Note

This preview.html is deliberately independent from the interactive builder. It is safe to show to stakeholders (no live data, no equipment capture, no AI calls).
